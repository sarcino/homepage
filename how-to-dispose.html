How to dispose...

Design an app which will help users to know how to sort trash.

DURATION: January and February 2022

TOOLS:
Figma
Adobe Illustrator
Google Workspace

Overview

This project was created as a part of the UX Google Design Certification Programme.

The goal is to help people with the sorting process by selecting the correct container. Users should be able to scan a barcode of the product and receive information on how they can dispose of trash correctly. The application should include a map with nearby disposal points.

Challenge

People don’t know where to throw away packages which can be sorted and recycled.

Ideation

My very first idea was very simple at the first sight. I wanted to use a database of all everyday used things most of us have in our household. Users could search for a particular thing and use a whisperer in the search field in the application. However, this idea wasn’t realistic and I decided to use a barcode scanner.

User Pain Points

Information

Googling the solution is time-consuming.

Based on competitive audit, existing websites have educational character with minimal specific information and are usually connected to the city which is operating disposal points.

Combined Materials

People tend to be lost when they need to throw away packages from combined materials.

Digital Wireframes & Low Fidelity Prototype

Not all the users will have the possibility to scan a barcode. I solved this problem by implementing the Cannot scan option which allows the user to select the material of the thing they would like to throw away.

I divided the possible options and information about sorting into 10 groups:

  • paper,
  • plastic,
  • metal,
  • glass,
  • compost,
  • wood,
  • chemicals,
  • furniture,
  • fabric,
  • and electronic.

Usability Study

Moderated usability study with 5 participants. Users tested the first version of the low fidelity prototype.

Accessibility

Some participants have concerns about readability because of insufficient color contrast. I changed dark grey color text to black color because the grey font didn’t meet a color contrast ratio of 4.5:1.

Meaning of illustrations shouldn’t be dependent on colors only. Therefore, I’m planning to add more descriptive illustrations. The real application should also include alternative texts.

Barcode scanning

Participants were not sure how the scanning process work. I added a description text on the homepage.

Learnings

This application works as a reminder that color should not be used as the only visual means of conveying information, indicating an action, prompting a response, or distinguishing a visual element. This is based on WCAG 1.4.1.
